# Contributing

## Flujo de trabajo

1. Crear rama desde `main`.
2. Hacer cambios pequeños y claros.
3. Ejecutar build y tests.
4. Crear Pull Request.
5. Solicitar revisión con Codex.

## Convenciones

- Usar Conventional Commits.
- No subir secretos.
- Mantener documentación actualizada.
- Usar DTOs para los endpoints.
